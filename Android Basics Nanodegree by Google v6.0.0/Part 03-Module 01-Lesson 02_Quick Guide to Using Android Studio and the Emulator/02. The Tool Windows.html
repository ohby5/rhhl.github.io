02. The Tool Windows

The Tool Windows

In addition to the main window pane where you add your code, Android Studio has a number of Tool Windows to task specific purposes - Project, Gradle, Terminal, etc.

Accessing (4 Options)

Option 1: The Tool Windows are the side panes in the left, right, and bottom edges of the main window:

Option 2: Tool Windows are also accessible from the top toolbar window of Android Studio via View > Tool Windows:

Option 3: Click on the Quick Access Menu in the bottom left corner of Android Studio (small grey square). Hovering over the icon pulls up a list of available Tool Window options.

Option 4: Lastly, in the main window, buttons with a number indicate the tool window can be opened by pressing ALT (COMMAND for Macs) + The Corresponding Number (Ex: 1:Project)

Using a Tool Window

Once a tool window is open, you can access its specific Settings from the cog icon in the top right corner:

Next to it is the the Hide icon which slides the window closed:

Here’s a list of commonly used tool windows. For the scope of this program, we’ll only focus on the ones marked with a *.

Tool Window Description
Android Monitor* Provides access to the Android debugging system - monitoring log output from a running application, taking screenshots and videos of the application, stopping a process and performing basic debugging task
Gradle* Shows the Gradle tasks that help the project build and run correctly. Don’t worry if you’re not familiar with Gradle for now; we will cover Gradle in more detail later in the course. In short, Gradle is the underlying system that helps wrap up Android projects and make sure they run properly.
Project* Shows all the files in the project. Double-clicking on a file opens it up to be edited.
Terminal* Provides direct access to a Command-Line interface of the computer that Android Studio is running on. For Windows, it uses the Command Prompt Interface. For Macs and Linux, it uses the Terminal.

The Command-Line Interface is the most direct ways to type commands to the computer.

Quick Guide to Command Line
TODO When developing, developers can make note of tasks that need to be completed by typing a TODO comment directly in the source file.

Android Studio will pattern match the TODO pattern and list all the TODOs in a project in this Tool Window to be reviewed and completed in the future.

Click for example TODO Image
Captures Captures a bug report from the connected device which contains device logs, stack traces, and other diagnostic information to help you find and fix bugs in your app. (Click for info to Enable)
Event Log Displays messages relating to events and activities performed within Android Studio. For example, it shows if a project was built successfully or if an application is running.
Favorites A variety of project items can be added to the Favorites list.

To add a file to Favorites , in the Project View > Right Click on the File > Add to Favorites

Click for screenshot

To add a method to Favorites, in the Structure Tool Window > Right Click on the Method > Add to Favorites

Click for screenshot
Gradle Console Displays information about what’s happening with the Gradle system (the underlying system that helps wrap up Android projects and make sure they run properly). The Gradle Console displays if there were any errors, warnings, or if it all ran successfully.
Messages Records output from the Gradle build system which can be used to help troubleshoot any problems when trying to compile the application.
Structure Shows high level view of the structure of whichever source file currently displayed in the editor. It includes information such as classes, methods and variables in the file. Selecting an item from the structure list will take you to that location in the source file in the editor window.
