04. Android Studio Updates and Understanding Release Notes

Version Updates

Google is constantly working on improving the IDE, adding improvements and fixing bugs. Periodic updates are pushed to Android Studio without requiring you to update your Android project. To manually check for updates, in the top toolbar of Android Studio go to Help > Check for Updates (On Mac Android Studio > Check for Updates)

Release Notes

Whether on your computer or mobile device, you’ve likely encountered the incessant prompt that “It’s time to update your software version!”

Once you’ve agreed to go through with the update, you’ll be presented several prompts confirming you agree to accept the changes in the latest version.

These changes often come in the Release Notes which are the documents shared with the users of the software about what changes were made in the release. There is no standard way to format Release Notes, but most of them contain generally the same information (more details found here).

Android Studio publishes its Release Notes on its Developer Site here.
There you can see what New features were added, what Changes were made, and Bug Fixes.
